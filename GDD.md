# 小资历神奇大冒险 - 游戏设计文档

## 1. 游戏概述
- 游戏名称：小资历神奇大冒险
- 类型：2D回合制肉鸽(Roguelike)塔爬游戏
- 平台：网页版 (HTML5 + JavaScript)
- 美术风格：像素风 + 暗黑哥特元素
- 目标用户：喜欢硬核肉鸽游戏的玩家

## 2. 核心玩法循环
[选择路径] → [战斗/事件/商店/休息] → [胜利/完成] → [奖励选择] → [下一层] → [死亡重开]

## 3. 地图系统
### 3.1 路径选择
每层开始时，玩家可从3条路径中选择1条前进：
- ⚔️ **战斗**：遭遇普通敌人
- 👹 **精英战**：遭遇精英敌人，奖励更丰富（每5层出现）
- 🏪 **商店**：购买物品强化自身
- 🏕️ **安全屋**：休息恢复或随机强化属性
- ❓ **未知事件**：随机事件，可能获得奖励或遭遇危险
- 👹 **BOSS战**：每10层的BOSS战（固定）

### 3.2 路径生成规则
- 每层生成3个不同的路径选项
- 每5层可能出现精英战
- 每10层固定为BOSS战

## 4. 战斗系统
### 4.1 回合机制
- 单动制：每个角色每回合只能执行1个动作
- 速度属性决定行动顺序
- 速度相同则随机排序

### 4.2 战斗布局
- 左右布局：玩家队伍在左，敌人在右
- 战斗场景：背景随区域变化

### 4.3 伤害计算
- 基础伤害 = 攻击力 - 敌方防御
- 暴击率/暴击伤害
- 元素克制（可选）

## 5. 角色系统
### 5.1 陈佳轩 - 【小资历】
- 定位：召唤系战斗
- 核心机制：画画召唤虚拟武器
- 玩家在战斗中选择画什么 -> 召唤对应的虚拟武器
- 初始武器类型：剑、弓、法杖、刀

### 5.2 队伍系统
- 初始：1人（陈佳轩）
- 上限：3人
- 获得方式：剧情解锁（特定层数触发）

### 5.3 角色属性
- 生命值(HP)
- 攻击力(ATK)
- 防御力(DEF)
- 速度(SPD)
- 暴击率(CRIT)
- 暴击伤害(CRIT_DMG)

## 6. 技能系统
### 6.1 技能获取
- 战斗胜利后奖励选择
- 商店购买技能卷轴
- 技能池：固定技能树 + 随机池

### 6.2 技能类型
- 攻击技能
- 防御技能
- 辅助技能
- 被动技能
- 召唤技能

### 6.3 技能稀有度
- 普通(白)
- 稀有(绿)
- 史诗(蓝)
- 传说(紫)

## 7. 宠物系统
### 7.1 宠物类型
- 参战型：作为额外战斗单位自动战斗
- Buff型：提供被动加成，不直接参战

### 7.2 宠物获得
- 击杀敌人随机掉落
- 蛋的形式获得

## 8. 圣人遗物系统
### 8.1 遗物类型
- 被动加成类：永久提升属性
- 一次性使用类：战斗中使用
- 特殊效果类：触发特殊效果

### 8.2 遗物获得
- 战斗胜利后奖励选择
- 商店购买遗物盒子
- 精英/BOSS掉落概率更高

## 9. 敌人系统
### 9.1 敌人类型
- 普通敌人：标准属性
- 精英敌人：属性翻倍，掉落率更高
- BOSS：特殊技能，高属性

### 9.2 区域主题
- 1-10层：地牢（阴暗、潮湿、监狱风格）
- 11-20层：深渊（黑暗、熔岩、洞穴风格）
- 21-30层：灵域（神圣、光明、仙界风格）

### 9.3 BOSS设计
- 第10层：地牢守卫者
- 第20层：深渊之主
- 第30层：灵域天王

## 10. 奖励系统
### 10.1 战斗奖励
战斗胜利后获得：
1. **金币**（必得）：基础金币 + 层数加成
   - 普通战：15 + 层数×3
   - 精英战：×2倍
   - BOSS战：×3倍

2. **二选一奖励**：
   - 技能：随机获得一个新技能
   - 圣人遗物：随机获得一个遗物
   - 可选择跳过不选

### 10.2 奖励概率
- 60%概率出现技能选项
- 40%概率出现遗物选项
- 至少保证1个可选奖励

## 11. 商店系统
### 11.1 商品类型
- 治疗药水：恢复生命
- 属性提升：永久增加攻击/防御/生命/暴击
- 技能卷轴：随机技能
- 遗物盒子：随机遗物

### 11.2 价格机制
- 基础价格 + 层数加成（每层+5%）

## 12. 安全屋系统
### 12.1 选项
- 休息：恢复30%最大生命
- 强化：随机提升一项属性（攻击+2/防御+2/生命+10/暴击+3%）

## 13. 事件系统
### 13.1 事件类型
- 神秘宝箱：可能获得金币或受伤
- 神秘泉水：恢复生命或获得药水
- 祭坛：消耗金币获得增益
- 神秘商人：低价购买物品
- 陷阱：必定受伤
- 游荡灵魂：帮助获得善缘

## 14. 死亡惩罚
- 硬核模式：死亡即重开
- 完全随机：重新从第1层开始
- 无任何继承

## 15. UI界面设计
### 15.1 主界面
- 开始游戏
- 继续游戏（存档）
- 设置

### 15.2 地图界面
- 显示当前层数
- 3个路径选项
- 玩家状态栏

### 15.3 战斗界面
- 玩家区域（左）
- 敌人区域（右）
- 状态栏（HP、行动顺序）
- 技能栏
- 伤害特效

### 15.4 奖励界面
- 金币显示（已获得）
- 二选一奖励卡片
- 跳过按钮

### 15.5 商店/安全屋/事件界面
- 弹窗式面板
- 选项按钮

## 17. 技术架构
### 17.1 文件结构
```
项目1/
├── GDD.md              # 本文档
├── index.html          # 游戏入口
├── css/
│   └── style.css       # 样式文件（响应式）
├── js/
│   ├── main.js         # 主程序入口
│   ├── Game.js         # 游戏主逻辑类
│   ├── Battle.js       # 战斗系统类
│   ├── Character.js    # 角色类
│   ├── Enemy.js        # 敌人类
│   ├── Skill.js        # 技能类
│   ├── Pet.js          # 宠物类
│   ├── Relic.js        # 圣人遗物类
│   ├── UI.js           # UI界面类
│   └── Data.js         # 数据配置
└── assets/             # 资源文件夹
    ├── images/         # 图片素材
    │   └── chenjiaxuan.png  # 主角立绘
    └── sounds/         # 音效素材
```

### 17.2 核心类设计
- Game: 游戏主循环、状态管理、奖励系统
- Battle: 回合管理、伤害计算
- Character: 角色属性、技能
- Enemy: AI行为、属性、类型
- UI: 渲染、交互处理、特效

## 18. 特效系统
### 18.1 战斗特效
- 攻击动画：角色卡片左右移动
- 受击特效：卡片闪烁 + 震动
- 伤害数字：浮动上升并消失
- 暴击特效：更大字体、金色显示
- 治疗特效：绿色数字

### 18.2 UI特效
- 按钮悬停：上浮 + 发光
- 卡片选中：边框高亮
- 面板出现：渐显效果
